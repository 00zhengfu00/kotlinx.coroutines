NonCancellable

object NonCancellable : AbstractCoroutineContextElement, Job (source)

A non-cancelable job that is always active. It is designed to be used with run builder to prevent cancellation of code blocks that need to run without cancellation.

Use it like this:

run(NonCancellable) {
    // this code will not be cancelled
}

onCompletion
fun onCompletion(handler: CompletionHandler): Registration

Registers completion handler. The action depends on the state of this job. When job is cancelled with cancel, then the handler is immediately invoked with a cancellation cause or with a fresh CancellationException. Otherwise, handler will be invoked once when this job is complete (cancellation also is a form of completion).
